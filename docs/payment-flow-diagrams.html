2. UCP Discovery — How Google Finds Your Store

Before any checkout happens, the AI Agent must discover what your store supports. You publish a UCP profile manifest at /.well-known/ucp that declares your capabilities, supported payment handlers, and API endpoints.

    Agent->>Merchant: GET /.well-known/ucp

    Agent->>Agent: Begin checkout flow

3. Checkout Session Lifecycle

A checkout session goes through distinct states. The AI Agent manages this lifecycle by calling your REST API endpoints. Each state transition is driven by user actions (selecting shipping, confirming payment, etc.). 

stateDiagram-v2
    [*] --> Created: POST /checkout-sessions
    Created --> Updated: PUT /checkout-sessions/:id
(address, shipping method)
    Updated --> Updated: PUT /checkout-sessions/:id
(tax recalculation, promo code)
    Updated --> Completing: POST /checkout-sessions/:id/complete
(with payment token)
    Created --> Completing: POST /checkout-sessions/:id/complete
(simple checkout)
    Completing --> Completed: Payment authorized ✅
    Completing --> Failed: Payment declined ❌
    Created --> Cancelled: POST /checkout-sessions/:id/cancel
    Updated --> Cancelled: POST /checkout-sessions/:id/cancel
    Failed --> Updated: User retries with different payment
    Completed --> [*]
    Cancelled --> [*]

4. Google Pay Token Flow (Gateway Encryption with MPGS)

When the user confirms payment, Google Pay generates an encrypted payment token. With gateway encryption (your setup), the token is encrypted for MPGS — your server never sees raw card data, maintaining PCI compliance. 

sequenceDiagram
    participant User as 👤 Consumer
    participant GPay as 💳 Google Pay
    participant UCP as 🟡 UCP Orchestrator
    participant Server as 🟣 Your Server
    participant MPGS as 🟠 MPGS Gateway
    participant Network as 🏦 Card Network

    User->>GPay: Tap "Pay with Google Pay"
    GPay->>GPay: User selects card / confirms biometric

    rect rgb(230, 244, 234)
        Note over GPay,MPGS: Gateway Encryption (PCI-safe)
        GPay->>GPay: Generate payment token
        GPay->>GPay: Encrypt token WITH MPGS public key
        GPay-->>UCP: Encrypted PaymentData token
    end

    UCP->>Server: POST /checkout-sessions/:id/complete
{paymentData: encryptedToken}
    Server->>Server: Extract token (DO NOT decrypt)
    Server->>MPGS: Forward encrypted token via MPGS API
PUT /api/rest/version/73/merchant/{id}/order/{oid}/transaction/{tid}

    rect rgb(255, 243, 224)
        Note over MPGS,Network: MPGS Processing
        MPGS->>MPGS: Decrypt token with private key
        MPGS->>MPGS: Extract PAN, expiry, cryptogram
        MPGS->>Network: Authorization request
        Network-->>MPGS: Approved / Declined
    end

    MPGS-->>Server: Payment result {status, authCode, gatewayTxnId}
    Server-->>UCP: Checkout session completed / failed
    UCP-->>User: "Order confirmed! 🎉" / "Payment failed"
Important: With gateway encryption, your server never handles raw card data. The token is encrypted specifically for MPGS using their public key. You simply pass it through. This means you do NOT need Level 1 PCI DSS compliance — SAQ A-EP is sufficient.

6. Complete End-to-End Sequence

The full journey from a user asking an AI agent to buy a product, through checkout, payment, and order confirmation. 

sequenceDiagram
    participant User as 👤 Consumer
    participant AI as 🤖 AI Agent
(Gemini/Search)
    participant UCP as 🟡 Google UCP
    participant Shop as 🟣 Merchant API
    participant GPay as 💳 Google Pay
    participant MPGS as 🟠 MPGS

    Note over User,MPGS: PHASE 1 — DISCOVERY
    User->>AI: "Buy a pair of Nike Air Max 90, size 10"
    AI->>Shop: GET /.well-known/ucp
    Shop-->>AI: UCP Profile (capabilities, payment handlers)

    Note over User,MPGS: PHASE 2 — SESSION CREATION
    AI->>UCP: Create checkout for product X
    UCP->>Shop: POST /checkout-sessions
{items: [{sku: "nike-am90", qty: 1, size: "10"}]}
    Shop-->>UCP: Session created {id, items, shipping_options, total}
    UCP-->>AI: Present options to user

    Note over User,MPGS: PHASE 3 — USER CONFIGURES ORDER
    AI->>User: "Ship to your default address?
Standard ($5.99) or Express ($12.99)?"
    User->>AI: "Express shipping to my home address"
    AI->>UCP: Update session with address + shipping
    UCP->>Shop: PUT /checkout-sessions/:id
{shipping: "express", address: {...}}
    Shop->>Shop: Recalculate tax + total
    Shop-->>UCP: Updated session {subtotal, tax, shipping, total}
    UCP-->>AI: Updated totals

    Note over User,MPGS: PHASE 4 — PAYMENT
    AI->>User: "Total: $118.98. Pay with Google Pay?"
    User->>AI: "Yes, confirm"
    AI->>GPay: Request payment token
    GPay->>User: Biometric / PIN confirmation
    User->>GPay: ✅ Confirmed
    GPay-->>AI: Encrypted payment token (for MPGS)

    Note over User,MPGS: PHASE 5 — COMPLETION
    AI->>UCP: Complete checkout with payment token
    UCP->>Shop: POST /checkout-sessions/:id/complete
{paymentData: encryptedToken}
    Shop->>MPGS: Forward token for payment
    MPGS->>MPGS: Decrypt → Authorize → Capture
    MPGS-->>Shop: SUCCESS {authCode: "ABC123"}
    Shop->>Shop: Create order, reserve inventory
    Shop-->>UCP: {status: "completed", orderId: "ORD-12345"}
    UCP-->>AI: Order confirmed

    Note over User,MPGS: PHASE 6 — CONFIRMATION
    AI->>User: "✅ Order confirmed!
Order #ORD-12345
Nike Air Max 90, Size 10
Express shipping to 123 Main St
Total: $118.98
Arriving in 2-3 business days"

    Note over User,MPGS: PHASE 7 — POST-PURCHASE (ASYNC)
    Shop-->>UCP: Webhook: Order shipped {tracking: "1Z999..."}
    UCP-->>AI: Notify user of shipment
